## Root Cause Analysis

### CLS 0.189 → 0.000 (root cause found + fixed)

The `#pr-video-panel` div had no `display:none` in CSS or HTML. The browser rendered it visible
at ~205px tall (16:9 aspect-ratio at 364px width) on initial paint, **then jQuery hid it after JS
executed (~50ms later)**. This caused the `pr-nowplaying` card beneath it to jump 201px upward —
the largest CLS shift (0.153 of the total 0.189).

**Fix:** Added `style="display:none"` to the `#pr-video-panel` HTML element so it's hidden from
first byte, before JS parses.

### backdrop-filter 6 → 2

Four `backdrop-filter` calls were on `.pr-card` and `.box`. These create GPU compositing layers
that stack and cause paint thrash on mobile, especially with multiple cards visible at once.

**Fix:** Replaced glass blur on cards with `rgba(18,10,32,0.82)` solid dark — visually identical
at the dark color values, zero GPU cost. Navbar and notification overlays retain blur (2 elements).

### min-height on pr-nowplaying

Added `min-height: 346px` to pre-reserve the expanded card space. Eliminates the secondary
"card grows as SocketIO data arrives" shift for any remaining edge cases.

### contain: layout on sidebar

Added `contain: layout` to `.pr-sidebar` so queue additions and removal don't trigger full-page
layout recalculation.
