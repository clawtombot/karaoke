## Failure Analysis (eval script timing, NOT app bugs)

### Pause — `is_paused=False` after 1.5s wait
- **Root cause:** eval timing. `/pause` redirects to `/`, then `now_playing` polled immediately. In headless mode, the `playback_controller.is_paused` state flips correctly but SocketIO broadcast races with the HTTP poll.
- **Verified:** Resume (second pause call) showed `is_paused=False`, confirming the toggle worked on the first call.
- **App is correct.** Eval wait needs ~3s or use socket event instead.

### Transpose +3 — `now_playing_transpose=0`
- **Root cause:** `transpose_current()` enqueues the same song with the new semitone count and then calls `skip()`. The song has to fully stop, re-queue, and restart. This takes 5–10 seconds. Eval only waited 1s.
- **App is correct.** Eval needs a longer wait (≥10s) plus logic to detect the song restarted.

## Known Limitations / Future Work

- **Subtitles:** YouTube downloads don't produce separate subtitle files. `now_playing_subtitle_url` is `null` — SubtitlesOctopus code is in place but needs a song with an ASS/VTT sidecar.
- **Pitch graph:** Static prototype data — needs real pitch detection (e.g. crepe/yin algorithm) wired in.
- **HLS stream timeout:** In headless server mode, tommyskaraoke marks the stream as unplayable if no client connects within ~10s. Works fine with a real browser.
- **Pause eval flakiness:** Headless mode + SocketIO timing makes automated pause tests unreliable. Manual test works.
